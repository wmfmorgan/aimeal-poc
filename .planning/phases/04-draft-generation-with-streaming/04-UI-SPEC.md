## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding, tag remove button spacing |
| sm | 8px | Compact element spacing, gap between preset buttons |
| md | 16px | Default element spacing, card internal padding (mobile) |
| lg | 24px | Section padding, card internal padding (desktop), grid gap |
| xl | 32px | Layout gaps between page sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (buttons, nav pills, stepper controls) — matches existing AppFrame and HouseholdPage pattern
- Glassmorphism card border-radius: `1.75rem` (28px) — matches `rounded-[1.75rem]` used across Phase 3

Source: existing `household-page.tsx` pattern (`min-h-[44px]`, `rounded-[1.75rem]`, `px-6 py-6`, `px-8 py-8`)

Source: `household-page.tsx` and `AppFrame.tsx` — replicate exact class patterns

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#faf9f8` (`--color-surface`) | Page background (via CSS gradient), outer shell |
| Secondary (30%) | `rgba(255,255,255,0.52)` / `rgba(255,255,255,0.72)` | Glassmorphism card containers, nav pill backgrounds |
| Accent (10%) | `#4A6741` (`--color-sage`) | Reserved for specific elements listed below |
| Destructive | `#803b26` | Error banners, error field borders, error text only |
| Ink | `#1f2520` (`--color-ink`) | Primary text, form field values |
| Muted | `#6f756c` (`--color-muted`) | Secondary/supporting text, placeholders, eyebrows |
| Accent warm | `#d7c8a3` (`--color-accent`) | Accent gradient, warm highlight layer (background only) |
| Sage deep | `#2f4430` (`--color-sage-deep`) | Active nav pills, section headings, chip text (active), hover states |

Accent (`#4A6741`) reserved for:
- Active nav pill background (current route)
- "Generate Your Plan" primary CTA button background
- Active preset button background (selected meal type or day count option)
- Skeleton card pulse glow tint (subtle, not full fill)
- "Generate Plan" submit button background on GenerationForm
- Active/selected chip fill (meal type preset buttons)

Accent NOT used for:
- Body text
- Card backgrounds
- Borders (use `rgba(74,103,65,0.2)` opacity variant only for input borders)
- Non-interactive decoration

Source: `globals.css`, `tailwind.config.ts`, `household-page.tsx` color patterns

## Component Inventory

### New components required in Phase 4

**GenerationForm** (`src/components/generation/GenerationForm.tsx`)
- Rendered inline on `/plan/:id` when plan has no meals (empty state)
- Container: `rounded-[1.75rem] bg-[rgba(255,255,255,0.72)] px-6 py-6 shadow-[var(--shadow-soft)] backdrop-blur-sm md:px-8 md:py-8`
- Eyebrow: `text-xs uppercase tracking-[0.26em] text-[var(--color-muted)]` → "Your weekly plan"
- Heading: `font-display text-[20px] font-semibold leading-snug text-[var(--color-sage-deep)]` → "Generate your plan"
- Body: `text-sm leading-7 text-[var(--color-muted)]` → "Choose which meals to include and how many days to plan."
- Meal type selector: three segmented preset buttons ("Dinner only" / "Lunch + Dinner" / "All three") — same segmented control pattern as cooking skill in HouseholdPage (`rounded-xl bg-[rgba(74,103,65,0.06)] p-1`, child buttons `flex-1 rounded-lg px-4 py-2 text-sm font-semibold min-h-[44px]`)
- Day count selector: stepper control (decrement button / count display / increment button), range 1–14, default 7; buttons are `rounded-full min-h-[44px] w-11` with sage border; count display is `text-base font-semibold text-[var(--color-ink)] w-10 text-center`
- Submit button: full-width on mobile, right-aligned on desktop; `rounded-xl bg-[#4A6741] px-6 text-white font-semibold text-sm tracking-wide min-h-[44px] transition-opacity disabled:opacity-60`; label: "Generate Your Plan →"
- Submit loading state: button disabled with label "Generating..." — no spinner, opacity 60%

**MealPlanGrid** (`src/components/generation/MealPlanGrid.tsx`)
- 7-column × 3-row CSS grid; columns = days (Mon–Sun), rows = meal types (Breakfast / Lunch / Dinner)
- On mobile: single-column stack, day label as section header above each day's meal cards
- Grid container: `grid grid-cols-1 gap-4 md:grid-cols-7`
- Row label column (desktop): `text-xs uppercase tracking-[0.26em] text-[var(--color-muted)]` at left edge
- Each cell: contains either a SkeletonMealCard (shimmer) or a MealCard (filled)

**SkeletonMealCard** (`src/components/generation/SkeletonMealCard.tsx`)
- Dimensions: match MealCard — `rounded-[1.75rem] bg-[rgba(255,255,255,0.72)] backdrop-blur-sm shadow-[var(--shadow-soft)]`
- Interior shimmer bars:
  - Title bar: `h-4 w-3/4 rounded-full`
  - Body bar 1: `h-3 w-full rounded-full mt-3`
  - Body bar 2: `h-3 w-5/6 rounded-full mt-2`
- Shimmer animation: CSS `animate-pulse` (Tailwind built-in) with fill color `bg-[rgba(74,103,65,0.08)]` — sage-tinted, calm, matches editorial aesthetic. No wave-sweep animation.
- Min height: `min-h-[7rem]` to reserve stable layout space before card fills in

**MealCard** (`src/components/generation/MealCard.tsx`)
- Container: `rounded-[1.75rem] bg-[rgba(255,255,255,0.72)] px-4 py-4 shadow-[var(--shadow-soft)] backdrop-blur-sm` — identical shell to SkeletonMealCard, swaps in on arrival
- Title: `text-sm font-semibold text-[var(--color-ink)] leading-snug`
- Description: `text-sm leading-7 text-[var(--color-muted)] mt-2`
- Phase 4: no interactive actions on card (inline edit, delete, detail view are Phase 5). Card is display-only.
- Animation on arrival: `transition-opacity duration-300` from opacity-0 to opacity-100 — subtle fade-in replacing the skeleton slot

**PlanReadyBanner** (`src/components/generation/PlanReadyBanner.tsx`)
- Shown after all SSE events received (stream complete)
- Container: `rounded-xl bg-[rgba(74,103,65,0.10)] px-4 py-3 text-sm leading-6 text-[var(--color-sage-deep)]` — matches success SubmitBanner pattern from household page
- Copy: "Your plan is ready. Meals will be editable in the next update." — reassures user, sets Phase 5 expectation
- Role: `role="status"` (non-disruptive, screen reader polite)
- Position: appears above the grid after stream completes; does not displace grid

**StreamErrorBanner** (`src/components/generation/StreamErrorBanner.tsx`)
- Shown if SSE stream fails mid-generation or returns error
- Container: `rounded-xl bg-[rgba(128,59,38,0.08)] px-4 py-3 text-sm leading-6 text-[#803b26]` — matches error SubmitBanner pattern
- Role: `role="alert"` (disruptive, screen reader assertive)
- Copy: "Something went wrong while generating your plan. [Try again →]" where "[Try again →]" is an inline button that resets to GenerationForm state
- Partial grid state: if some meals loaded before failure, show filled cards + remaining skeletons; do not wipe grid. Error banner appears above grid.

**DevPage** (`src/routes/dev-page.tsx`)
- Route: `/dev` — no auth gating (PoC)
- Nav label: "Dev" — added to `navItems` in `AppFrame.tsx`
- Page container: inherits AppFrame shell
- Two-section layout, stacked vertically with `space-y-8`

**LLMRequestsSection** (within DevPage)
- Container: `rounded-[1.75rem] bg-[rgba(255,255,255,0.72)] px-6 py-6 shadow-[var(--shadow-soft)] backdrop-blur-sm md:px-8 md:py-8`
- Eyebrow: "Developer tools"
- Heading: "LLM Requests"
- Body: "Last 10 generation calls. Expand any entry to inspect the full prompt and response."
- Entry list: `space-y-3` list of log row components
- Each log row: `rounded-[1.5rem] bg-white/60 px-4 py-4 md:px-5`
  - Header line: model name (`text-sm font-semibold text-[var(--color-ink)]`) + token count (`text-xs text-[var(--color-muted)]`) + timestamp (`text-xs text-[var(--color-muted)]`) in a flex row
  - Expandable section: prompt and response in `<pre>` tags with `text-xs leading-5 text-[var(--color-muted)] overflow-x-auto whitespace-pre-wrap break-words`
  - Expand toggle: `text-sm text-[var(--color-sage-deep)] hover:underline min-h-[44px] px-2` link-style button labeled "Show prompt/response" / "Hide"
- Empty state: `rounded-xl bg-white/60 px-4 py-4 text-sm leading-7 text-[var(--color-muted)]` → "No LLM calls yet. Generate a plan to see logs here."

**SpoonacularUsageSection** (within DevPage — Phase 4 placeholder)
- Container: same as LLMRequestsSection
- Eyebrow: "Developer tools"
- Heading: "Spoonacular Usage"
- Body: "Spoonacular API tracking will appear here in a future update."
- Content: placeholder paragraph only — no data, no live queries. Phase 6 replaces this content without layout refactor.

## Interaction States

### GenerationForm → Streaming → Grid transition

| State | What the user sees |
|-------|--------------------|
| Empty plan (no meals) | GenerationForm card centered on `/plan/:id` |
| Submit pressed | Form hides; 7×3 SkeletonMealCard grid appears immediately; submit button disabled during stream |
| SSE meal arrives | Corresponding skeleton slot fades in as MealCard (opacity transition 300ms) |
| All 21 meals received | PlanReadyBanner appears above grid; grid becomes visible and static |
| Stream error mid-flight | StreamErrorBanner appears above grid; partial cards remain visible; "Try again →" resets to GenerationForm |

### Day count stepper

| State | Behavior |
|-------|----------|
| Value 7 (default) | Decrement enabled (min 1), increment enabled (max 14) |
| Value 1 (minimum) | Decrement button `disabled`, opacity 40% |
| Value 14 (maximum) | Increment button `disabled`, opacity 40% |

### Preset meal type buttons

| State | Visual |
|-------|--------|
| Unselected | `bg-white/80 text-[var(--color-sage-deep)] border-[rgba(74,103,65,0.25)]` |
| Selected | `bg-[#4A6741] text-white border-[#4A6741]` (same as active chip pattern from Phase 3) |
| Only one option | All three buttons always visible; exactly one must be selected at all times |

### "Generate Your Plan →" button

| State | Visual |
|-------|--------|
| Default | `bg-[#4A6741] text-white` full opacity |
| Hover | `hover:opacity-90` |
| Disabled (streaming) | `disabled:opacity-60` |
| Loading | Button label: "Generating..." — no spinner |

## Accessibility

- All interactive elements: `min-h-[44px]` touch target (existing project pattern)
- PlanReadyBanner: `role="status"` (polite)
- StreamErrorBanner: `role="alert"` (assertive)
- SkeletonMealCard: `aria-label="Loading meal"` or `aria-hidden="true"` with wrapping `aria-live="polite"` region on MealPlanGrid
- Stepper buttons: `aria-label="Increase days"` / `aria-label="Decrease days"`
- Preset buttons: `aria-pressed={isSelected}` on each meal type option
- MealCard: semantic `<article>` element with `aria-label="{title}"`
- LLM log expand: `aria-expanded={isOpen}` on the toggle button

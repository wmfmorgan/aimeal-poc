Output ONLY one JSON object per line — no markdown, no wrapper array, no explanation:

Output exactly ${totalSlots} lines, one per meal slot. Each line must be a complete, valid JSON object.`;

**buildUserPrompt (extend existing with numDays + mealTypes):**
```typescript
function buildUserPrompt(config: {
  householdName: string;
  skillLevel: string;
  members: Array<{ name: string; allergies: string[]; avoidances: string[]; diet_type?: string | null }>;
  appliances: string[];
  numDays: number;
  mealTypes: string[];
}): string {
  const days = ["Monday","Tuesday","Wednesday","Thursday","Friday","Saturday","Sunday"].slice(0, config.numDays);
  const memberSummary = config.members
    .map(m => `- ${m.name}: allergies=[${(m.allergies ?? []).join(",")}], avoids=[${(m.avoidances ?? []).join(",")}]${m.diet_type ? `, diet=${m.diet_type}` : ""}`)
    .join("\n");
  return `Household: ${config.householdName}
Cooking skill: ${config.skillLevel}
Appliances: ${config.appliances.join(", ")}
Members:
${memberSummary}

Generate meals for these days: ${days.join(", ")}
Meal types per day: ${config.mealTypes.join(", ")}
Generate ${config.numDays * config.mealTypes.length} meal lines total.`;
}
```

**writeMealToDB helper (fire-and-forget — does not block SSE emission):**
```typescript
async function writeMealToDB(
  supabase: ReturnType<typeof createClient>,
  mealPlanId: string,
  line: string
): Promise<void> {
  try {
    const parsed = JSON.parse(line);
    // Strip rationale from SSE payload; store it in DB (D-06)
    await supabase.from("meals").insert({
      meal_plan_id: mealPlanId,
      day_of_week: parsed.day_of_week,
      meal_type: parsed.meal_type,
      title: parsed.title,
      short_description: parsed.short_description ?? null,
      rationale: parsed.rationale ?? null,
      status: "draft",
    });
  } catch {
    // DB write failure should not kill the stream
  }
}
```

**Main Deno.serve handler structure:**
```typescript
const openaiClient = new OpenAI({
  apiKey: Deno.env.get("XAI_API_KEY") ?? "",
  baseURL: "https://api.x.ai/v1",
});

Deno.serve(async (req) => {
  // CORS OPTIONS preflight
  if (req.method === "OPTIONS") {
    return new Response(null, {
      headers: {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "authorization, content-type",
      },
    });
  }

  // 1. Auth validation (replicate trpc/index.ts pattern exactly)
  const supabaseUrl = Deno.env.get("SUPABASE_URL") ?? "";
  const supabaseAnonKey = Deno.env.get("SUPABASE_ANON_KEY") ?? "";
  const authHeader = req.headers.get("Authorization") ?? "";
  const supabase = createClient(supabaseUrl, supabaseAnonKey, {
    global: { headers: authHeader ? { Authorization: authHeader } : {} },
  });
  let userId: string | null = null;
  if (authHeader.startsWith("Bearer ")) {
    const token = authHeader.slice("Bearer ".length);
    const { data } = await supabase.auth.getUser(token);
    userId = data.user?.id ?? null;
  }
  if (!userId) {
    return new Response("Unauthorized", { status: 401, headers: { "Access-Control-Allow-Origin": "*" } });
  }

  // 2. Parse + validate body
  let body: { householdId?: string; numDays?: number; mealTypes?: string[]; mealPlanId?: string };
  try {
    body = await req.json();
  } catch {
    return new Response("Invalid JSON body", { status: 400, headers: { "Access-Control-Allow-Origin": "*" } });
  }
  const { householdId, numDays = 7, mealTypes = ["breakfast","lunch","dinner"], mealPlanId } = body;
  if (!householdId || !mealPlanId) {
    return new Response("householdId and mealPlanId are required", { status: 400, headers: { "Access-Control-Allow-Origin": "*" } });
  }

  // 3. Fetch household (user's JWT scopes this via RLS)
  const { data: household, error: householdError } = await supabase
    .from("households")
    .select("id, name, cooking_skill_level, appliances, household_members(name, allergies, avoidances, diet_type)")
    .eq("id", householdId)
    .single();
  if (householdError || !household) {
    return new Response("Household not found", { status: 404, headers: { "Access-Control-Allow-Origin": "*" } });
  }

  // 4. Build prompts
  const totalSlots = numDays * mealTypes.length;
  const systemPrompt = buildSystemPrompt(totalSlots);
  const userPrompt = buildUserPrompt({
    householdName: household.name,
    skillLevel: household.cooking_skill_level ?? "intermediate",
    members: Array.isArray(household.household_members) ? household.household_members : [],
    appliances: Array.isArray(household.appliances) ? household.appliances : [],
    numDays,
    mealTypes,
  });

  // 5. LLM streaming call — NO response_format (D-05: incompatible with stream:true)
  const stream = await openaiClient.chat.completions.create({
    model: "grok-4-1-fast-non-reasoning",
    temperature: 0.7,
    stream: true,
    messages: [
      { role: "system", content: systemPrompt },
      { role: "user", content: userPrompt },
    ],
  });

  // 6. SSE ReadableStream — buffer tokens, emit one SSE event per completed NDJSON line
  let buffer = "";
  let fullResponse = "";
  let promptTokens = 0;
  let completionTokens = 0;

  const sseStream = new ReadableStream({
    async start(controller) {
      const encoder = new TextEncoder();
      try {
        for await (const chunk of stream) {
          const token = chunk.choices[0]?.delta?.content ?? "";
          buffer += token;
          fullResponse += token;

          const lines = buffer.split("\n");
          buffer = lines.pop() ?? "";

          for (const line of lines) {
            const trimmed = line.trim();
            if (!trimmed) continue;
            try {
              const parsed = JSON.parse(trimmed);
              // Emit SSE with only the client-facing fields (strip rationale per D-06)
              const ssePayload = JSON.stringify({
                day_of_week: parsed.day_of_week,
                meal_type: parsed.meal_type,
                title: parsed.title,
                short_description: parsed.short_description,
              });
              controller.enqueue(encoder.encode(`data: ${ssePayload}\n\n`));
              // Fire-and-forget DB write (does not block SSE emission)
              writeMealToDB(supabase, mealPlanId, trimmed);
            } catch {
              // malformed line — skip silently
            }
          }

          if (chunk.usage) {
            promptTokens = chunk.usage.prompt_tokens ?? 0;
            completionTokens = chunk.usage.completion_tokens ?? 0;
          }
        }
      } finally {
        // Persist LLM log after stream completes (D-14) — includes full response with rationale
        await supabase.from("llm_logs").insert({
          household_id: householdId,
          model: "grok-4-1-fast-non-reasoning",
          prompt: userPrompt,
          response: fullResponse,
          prompt_tokens: promptTokens,
          completion_tokens: completionTokens,
        });
        controller.enqueue(encoder.encode("data: [DONE]\n\n"));
        controller.close();
      }
    },
  });

  return new Response(sseStream, {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      "Access-Control-Allow-Origin": "*",
    },
  });
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→generate-draft | Untrusted POST body: householdId, mealPlanId, numDays, mealTypes |
| edge fn→Grok API | LLM receives household member data (name, allergies, avoidances) — prompt injection surface |
| edge fn→Supabase DB | Writes to meals and llm_logs using caller's JWT (anon key + Bearer) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Elevation of Privilege | generate-draft — unauthenticated access | mitigate | JWT validated via `supabase.auth.getUser(token)` at handler entry; 401 returned immediately if userId is null |
| T-04-06 | Tampering | generate-draft — householdId spoofing | mitigate | Household fetched using caller's JWT-scoped Supabase client; RLS on households restricts fetch to `user_id = auth.uid()`; non-owned household returns 404 |
| T-04-07 | Tampering | generate-draft — mealPlanId spoofing | mitigate | RLS on meals table: `meal_plan_id in (select id from meal_plans where user_id = auth.uid())`; writing meals to another user's plan fails at DB layer |
| T-04-08 | Tampering | LLM prompt injection via member allergies/name | accept | PoC: single-user households, user controls their own data; low risk for v1; future: sanitize member fields before interpolation |
| T-04-09 | Denial of Service | Unbounded generation requests per user | accept | PoC: no rate limiting; acceptable for single-user dev tool; future: per-uid rate limit at edge |
| T-04-10 | Denial of Service | SSE stream held open indefinitely | mitigate | Stream closes via `controller.close()` in finally block after all NDJSON lines emitted; `[DONE]` event signals client to stop reading |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> tRPC edge function | Untrusted route params and navigation-triggered reads enter the authenticated API surface |
| tRPC edge function -> Postgres via RLS | User-scoped data must not leak across meal plans owned by different users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | I | `mealPlan.get` | mitigate | Require `authedProcedure`, validate UUID input with Zod, and query through the caller JWT so RLS scopes rows to `auth.uid()` |
| T-05-02 | S | `mealPlan.latest` | mitigate | Resolve only from the authenticated user's records and return `{ id } | null` instead of arbitrary route strings |
| T-05-03 | T | `buildMealPlanSlots` | mitigate | Normalize only the allowed `MealType` values and generate explicit empty slots so malformed data cannot collapse the grid structure |
</threat_model>

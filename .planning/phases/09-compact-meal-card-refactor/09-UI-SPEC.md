## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-edge optical nudges, badge internal gaps, tooltip offset details |
| sm | 8px | Compact icon gaps, title-to-status separation, dense inline spacing |
| md | 16px | Default compact-card padding unit, action hit-area inset, stacked content rhythm |
| lg | 24px | Focused-card padding, card-to-card vertical rhythm in mobile/focused contexts |
| xl | 32px | Day-section spacing and larger grouped breaks around focused card contexts |
| 2xl | 48px | Major section breaks between plan header and card grid when cards are the primary surface |
| 3xl | 64px | Page-level spacing only; do not introduce inside the compact card system |

Exceptions: `44px` minimum hit target for all direct card actions and the full-card click target; focused cards may use `20px` internal padding only if needed to keep the shared compact payload readable without adding new content density

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.2 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.1 |

Implementation note: keep Phase 9 to these four sizes only. Dense cards use `Heading` for meal title, `Label` for status chips/tooltips, and `Body` only for confirmation or error text. Do not reintroduce long supporting copy into card bodies.

Visual hierarchy note: the meal title is the first-read anchor on every compact card, the status treatment is secondary supporting context, and icon actions are tertiary chrome that must stay visually subordinate to the title/flyout-entry behavior.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#faf9f8` | App background and the primary meal-planning canvas (source: `--color-surface`) |
| Secondary (30%) | `#ffffff` | Compact card surfaces, flyout panels, tooltip surfaces, and confirmation containers layered over the background |
| Accent (10%) | `#d7c8a3` | Reserved for compact-card hover wash, selected-card support tint, and tooltip emphasis only |
| Destructive | `#803b26` | Delete icon, destructive confirmation text, and destructive status treatment only |

Accent reserved for: clickable-card hover background tint, selected/active card support ring tint, tooltip emphasis accents, and subtle empty/error supporting highlights. Do not use accent for primary action fills, standard text, or every interactive control. Primary affirmative actions remain sage-driven (`#4A6741` / `#2f4430`) per the existing manual token system.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Open meal details` |
| Empty state heading | `No meal yet` |
| Empty state body | `Generate or regenerate this slot to add a meal. Details and favorites live in the flyout once a meal exists.` |
| Error state | `We couldn't update this meal. Try regenerating again, or open the meal details to review the current state.` |
| Destructive confirmation | `Delete meal`: `Remove this meal from the plan? You can regenerate this slot again afterward.` |

Additional copy rules for Phase 9:
- Remove `View details` as visible button copy; the card surface itself carries that action.
- Tooltip copy is verb-first and compact: `Regenerate meal`, `Delete meal`, `Saved to favorites`, `Open favorites`.
- Icon-only actions require persistent accessible names via `aria-label` even when a tooltip is present; tooltip text supplements the control but does not replace the label fallback.
- Dense cards must not render repeated `Breakfast`, `Lunch`, or `Dinner` labels when row or section context already supplies that information.
- Favorite copy leaves the compact card entirely and stays in the flyout/favorites surfaces.

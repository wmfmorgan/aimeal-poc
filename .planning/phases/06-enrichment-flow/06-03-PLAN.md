|----------|-------------|
| route-level selection state -> grid actions | Ephemeral client state controls which owned meals trigger enrichment |
| enriched recipe payload -> flyout and `/dev` presentation | Third-party recipe and usage data becomes user-visible content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | D | selection-mode UI | mitigate | Keep selection explicit, non-persistent, and scoped to visible filled cards so accidental batch actions stay reversible before submit |
| T-06-10 | T | flyout detail rendering | mitigate | Reuse the Phase 5 right-side shell and typed enriched fields instead of parsing raw JSON inside the component |
| T-06-11 | I | `/dev` presentation | mitigate | Display only usage metadata and cost counters, never auth headers, API keys, or internal request bodies |
| T-06-12 | D | live card updates | mitigate | Drive card-local loading and retry state from the route hook so one failed card cannot block the rest of the grid or the flyout |
</threat_model>

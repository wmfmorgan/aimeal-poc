# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Household Setup (/household).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (bespoke Tailwind primitives — no shadcn CLI) |
| Preset | not applicable |
| Component library | Radix (none installed) — compose from raw Tailwind only |
| Icon library | none declared — use inline SVG or Lucide if needed (already in React ecosystem) |
| Font — display | Newsreader (serif) via `font-display` Tailwind alias |
| Font — UI | Manrope (sans) via `font-sans` Tailwind alias |

**Source:** `tailwind.config.ts`, `src/styles/globals.css`, `src/routes/auth-page.tsx` (established Phase 2 patterns)

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline chip padding (x-axis only) |
| sm | 8px | Tag chip gap, member-row internal padding |
| md | 16px | Field-to-field spacing, label-to-input gap |
| lg | 24px | Section padding (card interior), group-to-group gap |
| xl | 32px | Section-to-section vertical rhythm |
| 2xl | 48px | Card-to-card gap, major surface breaks |
| 3xl | 64px | Page top/bottom padding |

Exceptions:
- Touch targets: all interactive elements (buttons, chips, expandable row toggle) minimum 44px height/width, sourced from `min-h-[44px]` pattern established in Phase 2.
- Allergen chips: 32px height (compact pill) with 12px horizontal padding — smaller than standard touch row, but finger-reach applies only when grouped together as a grid.

---

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Display | 48px / `text-5xl` (mobile) · 72px / `text-7xl` (md+) | 400 (regular) | 1.0 (`leading-none`) | Newsreader | AppFrame H1 (inherited, do not re-declare) |
| Heading | 20px / `text-xl` | 600 (semibold) | 1.25 (`leading-snug`) | Newsreader | Section headings: "Your Household", "Members", "Appliances" |
| Body | 16px / `text-base` | 400 (regular) | 1.75 (`leading-7`) | Manrope | Field descriptions, welcome nudge copy, member summary text |
| Label | 14px / `text-sm` | 600 (semibold) | — | Manrope | Form field labels, allergen chip labels, diet dropdown label |
| Caption | 12px / `text-xs` | 400 (regular) | — | Manrope | Eyebrow labels (`uppercase tracking-[0.26em]`), muted helper text, character count |

**Rule:** Two weights only — 400 (regular) and 600 (semibold). No 500/700.

**Source:** `auth-page.tsx` established `text-sm font-semibold` for labels and `text-base` for inputs; `AppFrame.tsx` established display/heading sizes.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#faf9f8` + `rgba(255,255,255,0.52–0.72)` glassmorphism layer | Page background (gradient from `globals.css`), outer AppFrame card |
| Secondary (30%) | `rgba(255,255,255,0.72)` | Household content card, expandable member row interior, appliances panel |
| Accent (10%) | `#4A6741` (sage) | Primary Save button fill, selected allergen chip fill, active state for diet dropdown border |
| Warm gold | `#d7c8a3` | Decorative — welcome nudge left-border stripe (4px), not for interactive states |
| Ink | `#1f2520` | All body text, field values |
| Muted | `#6f756c` | Placeholder text, captions, eyebrow labels, helper copy |
| Sage deep | `#2f4430` | Section headings, success banner text, secondary link text |
| Destructive | `#803b26` | Member delete confirmation text, field validation errors |
| Destructive surface | `rgba(128,59,38,0.08)` | Error banners, field-level error backgrounds |
| Success surface | `rgba(74,103,65,0.10)` | Save success banner background |

Accent (`#4A6741`) reserved for:
- "Save Household" button fill
- Selected/active allergen preset chips
- Input focus ring (`focus:ring-[#4A6741]/20 focus:border-[#4A6741]`)
- Active nav pill (inherited from AppFrame)

**Never use accent for:** muted text, borders on unselected states, decorative separators.

**Source:** `globals.css` CSS variables, `auth-page.tsx` color classes.

## Component Inventory

All components are bespoke Tailwind compositions. No shadcn CLI components are installed.

### 1. Page Container

The `/household` route renders inside `AppFrame` → `<main>`. No additional full-bleed wrapper needed.

```
<div class="space-y-8">
  <!-- welcome nudge (first-time only) -->
  <!-- household basics card -->
  <!-- members card -->
  <!-- appliances card -->
  <!-- save row -->
</div>
```

### 2. Glassmorphism Content Card

Reuse the exact pattern from Phase 2:

```
rounded-[1.75rem] bg-[rgba(255,255,255,0.72)] shadow-[var(--shadow-soft)] backdrop-blur-sm px-6 py-6 md:px-8 md:py-8
```

Used for: household basics section, members section, appliances section.

### 3. Section Heading

```
<p class="text-xs uppercase tracking-[0.26em] text-[var(--color-muted)]">Eyebrow</p>
<h2 class="font-display text-xl font-semibold leading-snug text-[var(--color-sage-deep)]">Section Title</h2>
```

### 4. Form Field

Same pattern as auth-page inputs:

```
<label class="block text-sm font-semibold text-[var(--color-ink)] mb-2">Label</label>
<input class="w-full rounded-xl border border-[rgba(74,103,65,0.2)] px-4 py-3 text-base
              bg-white/80 placeholder:text-[var(--color-muted)] outline-none
              focus:border-[#4A6741] focus:ring-2 focus:ring-[#4A6741]/20 transition-colors" />
<p class="mt-1 text-sm text-[#803b26]">Field error</p>  <!-- only when invalid -->
```

### 5. Cooking Skill Segmented Control

Three-option horizontal tab matching Phase 2 mode-switcher pattern:

```
<div class="flex rounded-xl bg-[rgba(74,103,65,0.06)] p-1">
  <button class="flex-1 rounded-lg px-4 py-2 text-sm font-semibold min-h-[44px]
                 bg-[#4A6741] text-white shadow-sm">  <!-- active -->
  <button class="flex-1 rounded-lg px-4 py-2 text-sm font-semibold min-h-[44px]
                 text-[var(--color-sage-deep)] hover:bg-white/60">  <!-- inactive -->
</div>
```

Labels: "Beginner" | "Intermediate" | "Advanced"

### 6. Allergen Preset Chips

Toggle pill pattern. Big 9 displayed as a wrapping chip grid:

```
<!-- Unselected -->
<button class="rounded-full border border-[rgba(74,103,65,0.25)] px-3 py-1.5
               text-xs font-semibold text-[var(--color-sage-deep)] bg-white/80
               hover:border-[#4A6741] transition-colors min-h-[32px]">
  Peanuts
</button>

<!-- Selected (toggled on) -->
<button class="rounded-full border border-[#4A6741] px-3 py-1.5
               text-xs font-semibold bg-[#4A6741] text-white min-h-[32px]">
  Peanuts
</button>
```

Big 9 preset labels (exact text, in this order):
Milk, Eggs, Fish, Shellfish, Tree Nuts, Peanuts, Wheat/Gluten, Soy, Sesame

### 7. Freeform Tag Input

For custom allergies and avoidances. Input with Enter-to-add, × to remove. Tag appearance:

```
<!-- Rendered tag -->
<span class="inline-flex items-center gap-1 rounded-full bg-[rgba(74,103,65,0.08)]
             px-3 py-1 text-xs font-semibold text-[var(--color-sage-deep)]">
  No mushrooms
  <button aria-label="Remove tag" class="text-[var(--color-muted)] hover:text-[#803b26]">×</button>
</span>

<!-- Tag input field -->
<input placeholder="Type and press Enter…"
       class="rounded-lg border border-[rgba(74,103,65,0.2)] px-3 py-2 text-sm
              bg-white/80 focus:border-[#4A6741] focus:ring-2 focus:ring-[#4A6741]/20 outline-none" />
```

### 8. Diet Type Dropdown

Native `<select>` styled to match the form field pattern. Fixed option list:

```
<select class="w-full rounded-xl border border-[rgba(74,103,65,0.2)] px-4 py-3 text-base
               bg-white/80 text-[var(--color-ink)] outline-none
               focus:border-[#4A6741] focus:ring-2 focus:ring-[#4A6741]/20 appearance-none">
  <option value="">Select diet type…</option>
  <option value="omnivore">Omnivore</option>
  <option value="vegetarian">Vegetarian</option>
  <option value="vegan">Vegan</option>
  <option value="pescatarian">Pescatarian</option>
  <option value="halal">Halal</option>
  <option value="kosher">Kosher</option>
  <option value="gluten-free">Gluten-Free</option>
  <option value="other">Other</option>
</select>
```

### 9. Appliances Input

Checkbox-chip pattern (multi-select). Same visual as allergen chips but with a checkmark on selection. Preset list:

Instant Pot, Air Fryer, Slow Cooker, Wok, Stand Mixer, Blender, Toaster Oven, Grill / BBQ, Sous Vide

Layout: wrapping flex row, same chip style as allergen chips.

### 10. Member List Row — Collapsed State

```
<div class="flex items-center justify-between rounded-xl bg-white/60 px-4 py-3 min-h-[52px]">
  <div>
    <p class="text-sm font-semibold text-[var(--color-ink)]">Alex</p>
    <p class="text-xs text-[var(--color-muted)]">Vegan · Peanuts, Shellfish</p>  <!-- summary -->
  </div>
  <div class="flex gap-2">
    <button class="text-sm text-[var(--color-sage-deep)] hover:underline min-h-[44px] px-2">Edit</button>
    <button class="text-sm text-[#803b26] hover:underline min-h-[44px] px-2">Remove</button>
  </div>
</div>
```

Summary format: `{diet_type} · {allergy1}, {allergy2}` — show up to 3 constraints, then "+ N more". If no constraints set, show "No dietary restrictions".

### 11. Member Row — Expanded State

Collapsed row expands in-place (no modal, no slide-over). Three sub-sections inside the expanded area, each separated by 24px vertical space:

1. **Allergies** — Big 9 chip grid + freeform tag input
2. **Avoidances** — Freeform tag input only + brief helper text: "Strong dislikes, not medical"
3. **Diet Type** — Dropdown

A "Done" text button collapses the row:

```
<button class="text-sm text-[var(--color-sage-deep)] hover:underline min-h-[44px]">Done</button>
```

### 12. Add Member Action

```
<button class="flex items-center gap-2 rounded-xl border border-dashed border-[rgba(74,103,65,0.3)]
               px-4 py-3 text-sm font-semibold text-[var(--color-sage-deep)]
               hover:border-[#4A6741] hover:bg-white/40 transition-colors w-full min-h-[52px]">
  + Add Member
</button>
```

### 13. Inline Delete Confirmation

Replaces the row's Edit/Remove buttons in-place when Remove is clicked:

```
<div class="flex items-center gap-3 text-sm">
  <span class="text-[#803b26]">Remove Alex?</span>
  <button class="font-semibold text-[#803b26] hover:underline min-h-[44px] px-2">Remove</button>
  <button class="text-[var(--color-muted)] hover:underline min-h-[44px] px-2">Cancel</button>
</div>
```

### 14. Primary Save Button

Full-width at bottom of page:

```
<button class="w-full rounded-xl bg-[#4A6741] px-6 text-white font-semibold text-sm tracking-wide
               transition-opacity disabled:opacity-60 min-h-[44px]">
  Save Household
</button>
```

Loading state: "Saving…" (disabled, opacity-60)

### 15. Success Banner

Appears in-place after save, above the Save button:

```
<p class="rounded-xl bg-[rgba(74,103,65,0.10)] px-4 py-3 text-sm leading-6
          text-[var(--color-sage-deep)]" role="status">
  Household saved. Your meal plans will reflect these preferences.
</p>
```

### 16. Welcome Nudge (first-time only)

Shown when no existing household is detected. Renders above the household basics card:

```
<div class="rounded-2xl border-l-4 border-[#d7c8a3] bg-[rgba(215,200,163,0.12)] px-5 py-4">
  <p class="text-sm font-semibold text-[var(--color-sage-deep)]">Set up your household to get personalised plans</p>
  <p class="mt-1 text-sm leading-6 text-[var(--color-muted)]">
    Tell us about your household's dietary needs and equipment. The AI uses this to tailor every plan it generates.
  </p>
</div>
```

## Page Layout

```
/household (single scrolling page, inside AppFrame > <main>)

  [Welcome Nudge]           — only on first visit (no household in DB)

  [Glassmorphism Card: Household Basics]
    Eyebrow: "Your Household"
    Heading: "Household Details"
    ─ Household Name (text input)
    ─ Cooking Skill (3-option segmented control: Beginner / Intermediate / Advanced)

  [Glassmorphism Card: Members]
    Eyebrow: "Who's Eating"
    Heading: "Household Members"
    ─ Member row (collapsed) × N
      └─ Expanded: Allergies, Avoidances, Diet Type
    ─ [+ Add Member] button

  [Glassmorphism Card: Appliances]
    Eyebrow: "Your Kitchen"
    Heading: "Available Appliances"
    ─ Appliance chip grid (multi-select)

  [Save Row]
    ─ Validation error summary (if any)
    ─ [Save Household] button (full-width)
    ─ Success banner (post-save, in-place)
```

Vertical rhythm between cards: `gap-8` (32px).

## Accessibility Contract

- All interactive elements meet 44×44px minimum touch target.
- Allergen chips are `<button type="button">` with `aria-pressed` for toggle state.
- Appliance chips are `<button type="button">` with `aria-pressed` for toggle state.
- Expanded member section uses `aria-expanded` on the Edit trigger button.
- Delete confirmation uses `role="alert"` on the confirmation text.
- Success banner uses `role="status"` (polite).
- Error banners use `role="alert"` (assertive).
- All form inputs have associated `<label>` elements (not placeholder-as-label).
- Native `<select>` for diet type (accessible by default, no custom dropdown in v1).
